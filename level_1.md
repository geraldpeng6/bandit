### 基本Linux命令 💻

本关卡需要了解以下基本命令：

- `ls`（list，列出）：列出目录中的文件和子目录
- `cd`（change directory，改变目录）：切换当前工作目录
- `cat`（concatenate，连接）：显示文件内容
- `file`：确定文件类型
- `du`（disk usage，磁盘使用）：估计文件空间使用量
- `find`：在目录层次结构中搜索文件

# Linux文件操作基础详解 📂

## ls命令全解析 🔍

`ls`命令是Linux中最常用的命令之一，用于列出目录内容。作为一名Linux初学者，掌握这个命令对于文件系统导航至关重要。

### ls命令的常用选项 ⚙️

| 选项 | 全称 | 功能 | 示例 |
|------|------|------|------|
| `-l` | long listing | 长格式显示，包含权限、所有者、大小等详细信息 | `ls -l` |
| `-a` | all | 显示所有文件，包括隐藏文件（以.开头） | `ls -a` |
| `-h` | human-readable | 以人类可读格式显示文件大小（KB、MB、GB） | `ls -lh` |
| `-t` | time | 按修改时间排序（最新的在前） | `ls -lt` |
| `-r` | reverse | 反向排序 | `ls -lr` |
| `-R` | recursive | 递归显示子目录内容 | `ls -R` |

### ls命令实用示例 🌟

```bash
# 基本用法 - 列出当前目录内容
ls

# 列出指定目录内容
ls /etc

# 长格式列出所有文件（包括隐藏文件）
ls -la

# 按大小排序（从大到小）
ls -lS

# 按时间排序并以人类可读格式显示
ls -lth
```

### 理解ls长格式输出 📊

当使用`ls -l`命令时，输出格式如下：
```
-rw-r--r-- 1 bandit0 bandit0 33 May 7 2020 readme
```

这个输出包含以下信息：
- `-rw-r--r--`：文件权限（类型和访问权限）
- `1`：硬链接数量
- `bandit0`：文件所有者
- `bandit0`：文件所属组
- `33`：文件大小（字节）
- `May 7 2020`：最后修改时间
- `readme`：文件名

## cat命令详解 📄

`cat`命令（concatenate的缩写）是另一个基本但功能强大的Linux命令，主要用于查看文件内容。

### cat命令的常用选项 ⚙️

| 选项 | 功能 | 示例 |
|------|------|------|
| `-n` | 显示行号 | `cat -n file.txt` |
| `-b` | 显示非空行的行号 | `cat -b file.txt` |
| `-s` | 压缩连续的空行为一行 | `cat -s file.txt` |
| `-E` | 在每行末尾显示$符号 | `cat -E file.txt` |

### cat命令实用示例 🌟

```bash
# 基本用法 - 显示文件内容
cat readme

# 显示带行号的文件内容
cat -n readme

# 创建新文件并输入内容（按Ctrl+D结束）
cat > newfile.txt

# 将多个文件内容合并到一个新文件
cat file1 file2 > combined.txt

# 将文件内容追加到另一个文件
cat file1 >> file2
```

## Linux文件系统结构 🌲

Linux文件系统遵循文件系统层次结构标准(FHS)，这是一种组织文件和目录的约定。了解这个结构对于在Linux系统中导航非常重要。

### 主要目录及其用途 📁

| 目录 | 用途 | 重要性 |
|------|------|--------|
| `/` | 根目录，所有其他目录的起点 | 核心 |
| `/bin` | 基本命令二进制文件 | 系统运行必需 |
| `/etc` | 系统配置文件 | 系统配置 |
| `/home` | 用户主目录 | 用户数据 |
| `/tmp` | 临时文件 | 临时存储 |
| `/usr` | 用户程序和数据 | 应用程序 |
| `/var` | 可变数据（日志等） | 系统运行数据 |

### 文件路径表示法 🛣️

在Linux中，有两种表示文件路径的方式：

1. **绝对路径**：从根目录开始的完整路径
   ```bash
   /home/bandit0/readme
   ```

2. **相对路径**：相对于当前工作目录的路径
   ```bash
   # 如果当前在/home/bandit0目录
   readme  # 指向/home/bandit0/readme
   ```

### 特殊路径符号 🔣

| 符号 | 含义 | 示例 |
|------|------|------|
| `.` | 当前目录 | `./script.sh` |
| `..` | 上一级目录 | `cd ..` |
| `~` | 当前用户的主目录 | `cd ~` |
| `/` | 根目录或路径分隔符 | `/home/user` |

## 文件操作实用技巧 💡

1. **使用Tab键自动补全**：输入文件名的前几个字符后按Tab键，系统会自动补全文件名
2. **使用通配符**：`*`匹配任意多个字符，`?`匹配单个字符
   ```bash
   # 列出所有.txt文件
   ls *.txt
   ```
3. **组合命令**：使用管道符`|`将多个命令组合使用
   ```bash
   # 查找包含"password"的行
   cat readme | grep password
   ```
4. **重定向输出**：使用`>`和`>>`将命令输出保存到文件
   ```bash
   # 将ls命令的输出保存到文件
   ls -la > directory_contents.txt
   ```

## 文件权限基础 🔒

Linux文件权限系统是安全性的重要组成部分。每个文件都有三组权限：所有者(u)、组(g)和其他用户(o)。

### 权限类型

- **r (read)**：读取权限，数值为4
- **w (write)**：写入权限，数值为2
- **x (execute)**：执行权限，数值为1

### 查看文件权限

使用`ls -l`命令查看文件权限：
```
-rw-r--r-- 1 bandit0 bandit0 33 May 7 2020 readme
```

权限部分`-rw-r--r--`的解释：
- 第1位：文件类型（`-`表示普通文件，`d`表示目录）
- 第2-4位：所有者权限（`rw-`表示可读可写不可执行）
- 第5-7位：组权限（`r--`表示只读）
- 第8-10位：其他用户权限（`r--`表示只读）
